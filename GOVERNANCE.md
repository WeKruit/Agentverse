# Governance

## Current Model: BDFL (Benevolent Dictator For Life)

Agentverse and the EACP protocol are in early development. All protocol decisions are made by the maintainer.

This is deliberate. Early-stage protocols need velocity over consensus. The governance model will evolve as the community grows.

## Decision Process

1. **Protocol changes**: Proposed via GitHub Issues tagged `protocol`. Discussed publicly. Maintainer decides.
2. **Code changes**: Standard PR process. Maintainer reviews and merges.
3. **Security issues**: Report privately via security@agentverse.app (or GitHub Security Advisories). Handled with urgency.

## Evolution Plan

| Contributors | Governance Model |
|-------------|-----------------|
| 1-10 | BDFL (current) |
| 10-50 | Core team of 3-5 trusted contributors with BDFL tiebreaker |
| 50-200 | RFC process for protocol changes (like Nostr NIPs) |
| 200+ | Independent foundation or Linux Foundation project |

## Principles

1. **All discussion is public.** No private channels for protocol decisions.
2. **Code speaks louder than proposals.** Working implementations are prioritized over theoretical designs.
3. **Security is non-negotiable.** No feature ships if it weakens the security model.
4. **Backward compatibility matters.** Breaking changes require a migration path.

## Code of Conduct

Be respectful. Be constructive. Focus on the work.
